# Contributing

## Branching

- Use feature branches from `main`.
- Prefer small pull requests with focused scope.

## Commit style

- Conventional commits enforced with commitlint.
- Example: `feat(products): add url synced filters`

## Pull request checklist

- [ ] `pnpm lint`
- [ ] `pnpm test:unit`
- [ ] `pnpm build`
- [ ] Updated docs when architecture/behavior changed
- [ ] Verified search/filter/pagination/detail manually
